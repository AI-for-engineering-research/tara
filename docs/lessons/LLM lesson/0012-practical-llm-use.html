Lesson 0012 · ~15 min

Practical LLM use from the mechanism

Core idea: Good LLM use comes from managing context, grounding, decoding, and verification.

What to remember

  • Prompting shapes context.
  • Grounding helps when facts matter.
  • Decoding changes output behavior.
  • Verification matters for high-stakes tasks.

Why it matters

This turns the mechanism-level model into practical prompting and system-design habits.

Mental model

Bad answer? Improve context, improve grounding, adjust selection, or add verification.

Checkpoint

  1. If an output is vague, what should you change first?
  2. If an output is wrong, what should you add?
  3. If an output is inconsistent, what decoding change might help?
