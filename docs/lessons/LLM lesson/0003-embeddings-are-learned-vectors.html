Embeddings are learned vectors

Core idea: Tokens are turned into learned vectors that place similar or useful patterns in related representational space.

What to remember

  • Embeddings are not labels; they are numeric representations.
  • The model computes with vectors, not raw text.
  • Meaning emerges partly from how representations are used in later layers.

Why it matters

Embeddings are the bridge from token IDs to model computation. They let the model treat tokens with related roles or usage patterns in useful ways.

Mental model

Text → tokens → token IDs → embeddings → transformer layers.

Checkpoint

  1. Why does the model need embeddings instead of plain token IDs?
  2. Why is an embedding more than a lookup label?
