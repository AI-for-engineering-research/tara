Using Backlog + pi to rebuild my research repo

July 21, 2026

This week I have been using Backlog to organize the process of creating a new repository for my low-aromatic fuel research project. My old repo got messed up because I pushed large files to GitHub, so I have been rebuilding the project in a cleaner, more controlled way.

My workflow has been: first I create a rough draft of a task, then I ask pi to refine the draft and turn it into a clearer Backlog task. If the task still needs more detail, I improve it. Once the task is ready, I ask pi to execute it. After each completed task, I ask pi to keep updating a running summary document: doc-1 - Completed-Tasks-Summary.md.

This has made the rebuild feel more systematic. Instead of trying to fix everything at once, I can draft, refine, execute, and document each task as a separate step.
