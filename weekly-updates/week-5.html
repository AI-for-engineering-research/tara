Updates

I wanted to run my refinery process cost model with a lot more crude oils because I was only using three in the past. I had also been assuming a US Gulf Coast refinery location, but costs vary depending on plant location.

I used deep research to find sources for refinery data, and found that GlobalData has information on refinery locations and capacities (MIT subscribes to it).

Next, I downloaded all available crude assays from BP online (35 crudes), and asked ChatGPT to assess which crudes are most likely used at each refinery. It generated this spreadsheet: refinery_crude_slate_inference_bp_matches.xlsx. I also saved the underlying Deep Research PDF here: Inferred Refinery Crude Slates and BP Assay Matches-2.pdf.

Crude–refinery matching logic

  1. Use refinery type and Nelson Complexity Index to infer crude tolerance.
  2. Adjust target API, sulphur, and acidity by region/country/import basin.
  3. Rank BP assays using API/sulphur/acidity distance plus a geographic/basin preference.

Then I used pi to build a new Python script that runs the refinery process cost model across all these refineries. The script outputs a spreadsheet of the main results (cost of treatment) for each refinery: all_refineries_top_match_treating_costs 2.numbers.

  • Applies plant location factors to account for construction cost differences across locations
  • Uses spatial wind electricity price data
  • Uses an appropriate crude slate for each refinery (from the matching step)
